### 11:00 — 修复更新用户失败的 Bug

- **类型：** Bug修复
- **范围：**
  - `backend/.../application/UserApplicationService.java`
  - `frontend/src/pages/UserEditPage.tsx`
  - `frontend/src/api/client.ts`
- **摘要：** 修复三个连锁问题导致前端更新用户显示 "Failed to update user"
- **根因分析：**
  1. `UserEditPage` 始终发送 `status` 字段（即使未修改），导致 ACTIVE→ACTIVE 被状态机拒绝
  2. `UserApplicationService.updateUser()` 未跳过"相同状态转换"（应为 no-op）
  3. `client.ts` 错误拦截器未从非 2xx 响应中提取结构化错误，真实错误被吞没
- **修复：**
  1. 后端：`target == user.getStatus()` 时跳过状态转换校验
  2. 后端：补充 `username` 字段的更新处理
  3. 前端：仅发送与原值不同的字段（diff 比较）
  4. 前端：错误拦截器从 `error.response.data` 提取 `{code, message}` 构造 `ApiError`
- **验证：**
  - Backend: `./mvnw test` 全绿，`./mvnw compile` 通过
  - Frontend: `tsc --noEmit` 通过，`npm run build` 成功
